# Local AI Studio

## 📖 项目简介

Local AI Studio 是一款功能强大的 Windows 本地 AI 工作室应用程序，基于 WPF 和 .NET 8 开发，采用现代化 UI 设计，支持毛玻璃效果。该程序集成了 Ollama 本地大模型管理、远程桌面控制、硬件监控、网站服务器托管等多项功能，可作为免安装的便携式工具使用。

本项目采用 MVVM 架构模式开发，所有服务模块化设计，便于维护和扩展。程序支持单文件发布，无需安装 .NET 运行时即可在 Windows 10/11 x64 系统上独立运行。

## ✨ 核心功能

### 1️⃣ Ollama 模型管理

程序提供完整的 Ollama 本地大模型管理功能。用户可以通过程序界面检测系统是否已安装 Ollama，查看已安装的模型列表，并选择要启动的模型。程序支持自动检测 Ollama 安装路径，自动获取本机 IPv4 地址，并提供一键部署功能。对于网络环境特殊的用户，程序还支持配置国内镜像源，确保 Ollama 能够顺利下载和更新。

在模型选择方面，程序采用现代化的列表界面展示已安装的模型，每个模型旁边配有圆形选择框，方便用户勾选要启动的模型。勾选完成后，点击下一步按钮即可启动选中的模型。程序会自动执行 `ollama serve` 或 `ollama run` 命令，并在启动过程中显示实时状态。

### 2️⃣ AI 聊天界面

程序内置类似 DeepSeek 的 AI 聊天界面，用户可以直接在程序中与本地部署的大模型进行对话。聊天界面支持流式输出，AI 的回复会实时显示，无需等待完整响应。输入框支持回车发送，界面设计简洁美观，提供良好的用户体验。

### 3️⃣ 硬件监控面板

硬件监控面板提供实时性能数据展示，包括 CPU、GPU、内存、存储和网络占用率。程序每秒更新一次数据，并使用动态曲线图展示历史趋势。蓝色的系统曲线与橙色的 AI 进程曲线重叠显示，方便用户对比系统整体负载与 AI 模型占用情况。

面板同时显示本机 IPv4 地址、内网穿透地址和当前用户名等信息，方便用户了解网络连接状态。所有数据通过 Windows Performance Counter API 获取，确保数据的准确性和实时性。

### 4️⃣ 远程桌面控制

程序支持远程桌面功能，被控端开启程序后可接受远程连接。远程控制采用 VNC 协议实现，程序提供简化版的屏幕共享支持。用户只需输入远程设备的连接地址，即可建立远程控制会话。

远程桌面功能支持跨局域网访问，可通过内网穿透服务实现公网连接。程序内置连接权限管理，被控端用户可以随时查看当前连接状态，必要时可强制断开所有远程连接。

### 5️⃣ 网站服务器托管

程序内置轻量级 HTTP 服务器，支持同时托管多个本地网站。用户可以通过界面添加网站，指定网站根目录和监听端口。服务器支持静态文件服务、目录浏览、MIME 类型自动识别和 CORS 跨域配置。

每个托管的网站都会在界面中显示状态、访问地址和访问统计。用户可以一键复制访问链接，或直接在浏览器中打开预览。内网穿透功能与网站服务器深度集成，支持为每个网站配置独立的公网访问地址。

### 6️⃣ 内网穿透配置

内网穿透功能基于 frp（Fast Reverse Proxy）实现，程序自动生成配置文件并管理 frpc 进程。用户只需填写服务器地址、端口、账号密码等信息，程序即可自动建立内网穿透通道。

配置界面包含 IPv4 地址自动检测、端口设置、用户名密码验证和自定义子域名配置。程序会验证输入信息的格式，确保配置正确后才会启动穿透服务。内网穿透支持与远程桌面、硬件监控和网站服务器等多个模块联动。

### 7️⃣ 硬件检测与远程访问

程序提供丰富的硬件检测功能，包括摄像头画面捕获、麦克风录音、WiFi 连接信息和 USB 设备列表。所有硬件访问都通过 RESTful API 提供，远程客户端可以通过 HTTP 请求获取实时数据。

摄像头监控使用 Windows AVI Capture API 实现，支持定时快照和实时视频流。麦克风功能实现对讲模式，远程用户说话的声音会通过主机扬声器播放。WiFi 信息通过 netsh 命令获取，展示 SSID、信号强度、连接状态等数据。

### 8️⃣ 守护进程与自更新

程序内置守护进程服务，监控 Ollama、frpc 等关键进程的运行状态。当检测到进程意外退出时，守护服务会自动尝试重启，确保服务的连续性。守护进程内置重启次数限制和冷却时间，防止因程序本身问题导致的死循环。

自更新功能支持从 GitHub Releases 检查新版本，自动下载更新包并替换程序文件。更新过程采用批处理脚本实现，确保文件替换的原子性，用户无需手动操作即可完成版本升级。

## 🛡️ 安全特性

### 加密存储

程序使用 Windows DPAPI（Data Protection API）对敏感配置进行加密存储。账号密码、内网穿透令牌、API 访问密钥等信息经过加密后保存到本地配置文件，即使文件被他人获取也无法直接读取。DPAPI 的作用域设置为当前用户，确保数据只能在本机被解密。

### 密码哈希

用户密码使用 BCrypt 算法进行哈希存储。BCrypt 是专门为密码存储设计的哈希函数，内置盐值（salt）和可配置的工作因子，能有效抵御彩虹表攻击和暴力破解。每次验证密码时，程序会使用计时安全的比较函数，防止时序攻击。

### 身份验证

远程访问 API 支持多种身份验证方式。Basic Auth 通过 HTTP Authorization 头传递用户名密码；Token 认证使用自定义的 JWT 风格令牌，包含签名和有效期验证。只有通过验证的请求才能访问受保护的资源。

### 隐私保护

程序非常重视用户隐私保护。当摄像头、麦克风或远程桌面等敏感功能被激活时，系统托盘区域会显示彩色图标提示用户。图标颜色对应不同的访问类型，用户可以随时点击查看当前状态。

被控端用户拥有完整的连接控制权。程序内置连接权限管理，所有入站连接都需要经过被控端批准或拒绝。用户在收到连接请求时可以查看请求方的 IP 地址和连接类型，然后决定是否接受。用户也可以随时撤销已批准的权限或强制断开当前所有连接。

## 🎨 界面设计

### 视觉风格

程序采用蓝色（#2A6DFF）配白色的现代化设计主题。主色调使用蓝色渐变，营造专业可信的视觉感受。所有卡片元素采用大圆角设计（24px），配合柔和的阴影效果，呈现层次分明的界面结构。

### 毛玻璃效果

窗口采用毛玻璃视觉效果，背景使用半透明渐变，边缘配合轻微的阴影和模糊效果。标题栏和内容区域的边界使用圆角处理，整体视觉效果现代且统一。

### 动画系统

所有交互元素都配备了精心设计的动画反馈。按压动画使按钮在点击时有轻微的缩放下沉效果（ScaleTransform 至 0.95），释放时以弹性缓动函数恢复。悬浮动画在鼠标悬停时改变元素亮度和阴影，增强交互感。所有动画时长统一设置为 150-250 毫秒，使用 EaseOut 或 BackEase 缓动函数确保动画流畅自然。

### 控件样式

程序定义了统一的控件样式资源字典，包括主按钮（蓝色渐变背景、白色文字）、次要按钮（灰色背景）、危险操作按钮（红色背景）、圆形开关（蓝色滑块、弹性动画）、圆形复选框（蓝色填充、白色对勾）、现代文本输入框（16px 圆角、聚焦高亮）和卡片容器（24px 圆角、柔和阴影）。

### 响应式布局

主界面采用响应式网格布局，内容区域能够根据窗口大小自动调整。侧边导航栏固定宽度，主要内容区域占据剩余空间。窗口支持拖拽调整大小，最小尺寸限制为 900x600 像素，确保界面在任何情况下都能正常显示。

## 🔧 构建与部署

### 环境要求

构建本项目需要安装以下软件：

.NET 8 SDK 是必需的构建工具，可从微软官方网站下载安装。安装完成后，可以在命令行中执行 `dotnet --version` 验证安装是否成功。程序本身运行时不需要安装 .NET 运行时，因为发布版本采用自包含模式，所有运行时依赖都已打包进单个 exe 文件。

### 构建步骤

项目根目录包含 PowerShell 发布脚本 `publish.ps1`，使用前请确保已安装 .NET 8 SDK。以下是详细的构建流程：

首先打开 PowerShell 终端，进入项目根目录。如果这是首次构建，建议先执行清理操作：

```powershell
.\publish.ps1 -Clean
```

这将删除之前的构建产物，确保本次构建从干净的状态开始。

接下来执行发布命令：

```powershell
.\publish.ps1
```

脚本会自动执行以下操作：检查 .NET SDK 版本，还原 NuGet 包，编译项目，发布单文件 exe。发布完成后，程序会显示输出文件路径和大小，并询问是否立即运行。

如果需要调试模式构建（带调试符号、编译更快）：

```powershell
.\publish.ps1 -Debug
```

发布产物位于 `publish` 目录下，主要文件为 `LocalAIStudio.exe`，文件大小通常在 80-120MB 之间（取决于依赖项）。

### 可选依赖项

虽然程序可以独立运行，但以下可选依赖项能提供更完整的功能体验：

frpc.exe（frp 客户端）用于内网穿透功能，可从 GitHub frp releases 页面下载与系统架构匹配的版本，下载后放置在与 LocalAIStudio.exe 同一目录下。

UltraVNC WinVNC 用于增强远程桌面功能，可从 uvnc.com 官网下载安装包。安装后将 winvnc4.exe 复制到程序的 AppData 目录下即可启用完整的 VNC 服务功能。

NAudio 库已在项目中通过 NuGet 引用，无需额外安装。该库用于音频播放和录制功能。

System.Management 用于 WMI 查询，获取系统硬件信息。该库同样已通过 NuGet 引用。

## 📖 使用指南

### 首次启动

首次启动程序后，界面会显示主页面，包含四个主要功能模块的入口：硬件显示、AI调用、远控电脑和网站服务器。用户可以根据需要导航到相应的功能页面进行配置。

### Ollama 配置

在 Ollama 设置页面，程序会自动检测系统是否已安装 Ollama。如果检测到已安装，页面会显示绿色对勾和"Ollama 已就绪"的提示。如果未安装，页面会显示官方下载地址和一键部署按钮（国内用户可配置镜像源）。

Ollama 启动后，切换到"模型管理"页面即可查看已安装的模型列表。勾选要启动的模型，点击启动按钮即可。模型启动后，切换到"AI调用"页面即可开始对话。

### 内网穿透配置

在内网穿透配置页面，用户需要填写服务器连接信息和认证凭据。IPv4 地址会自动检测并填充，用户只需填写最后一端数字。用户名和密码为必填项，程序会验证输入格式。

开启内网穿透开关后，可以为服务配置自定义子域名。保存配置后，点击"启动服务"按钮即可建立穿透通道。服务启动后，页面会显示局域网访问地址和公网访问地址。

### 网站托管

在网站服务器页面，点击"添加网站"按钮可以添加新的托管网站。需要指定网站名称、网站文件所在目录和监听端口。添加完成后，点击启动按钮即可开始托管。

托管启动后，用户可以一键复制访问链接，或直接在浏览器中打开预览。每个网站都会显示访问次数统计，方便用户了解访问情况。

### 远程桌面

远程桌面功能用于远程控制其他安装了 Local AI Studio 的计算机。在"远控电脑"页面输入远程设备的地址，点击连接按钮即可建立连接。

作为被控端时，用户可以设置是否接受入站连接请求。收到连接请求时，程序会弹出通知，用户可以选择批准或拒绝连接。在任何时候，用户都可以点击"强制断开"按钮终止所有远程连接。

### 安全设置

程序的安全设置位于设置页面，包含开机自启动、息屏功能、静默启动和自适应模式等选项。所有敏感配置都经过 DPAPI 加密存储在本地的 config.json 文件中。

开启息屏功能后，点击关机按钮会让系统进入睡眠状态而非真正关机。只有物理电源键可以唤醒，这为共享电脑的用户提供了额外的隐私保护。

### 守护与更新

在服务守护与更新页面，用户可以查看所有受监控进程的状态，配置自适应模式的资源阈值，以及检查程序更新。

自适应模式会在系统资源紧张时自动降低 AI 进程的优先级，确保系统保持响应。资源阈值可以在界面中自定义调整，默认设置为系统 CPU 超过 80% 或 AI 进程 CPU 超过 50% 时触发调整。

## ⚠️ 注意事项

程序需要管理员权限才能完整支持某些功能，如开机自启动的注册表写入、息屏功能的系统调用等。建议以管理员身份运行程序，或在需要时授予相应的权限。

内网穿透功能需要配合 frp 服务端使用。用户可以使用公开的 frp 服务，也可以自建 frp 服务端。程序默认配置为连接公共测试服务器，实际使用时需要替换为自建或可信的服务端地址。

远程桌面功能建议仅在可信的网络环境中使用。虽然程序内置了密码验证和连接审批机制，但远程控制仍存在一定的安全风险。对于敏感操作，建议配合 VPN 或其他安全通道使用。

程序的所有配置数据都存储在当前用户的 AppData 目录下，包括注册表 `HKCU\Software\LocalAIStudio` 和程序根目录下的 `config.json`。卸载程序时这些数据不会自动清除，如需彻底清理请手动删除。
